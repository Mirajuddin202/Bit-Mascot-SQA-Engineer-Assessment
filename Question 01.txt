3.Explain why each test case is important.
TC-001: Valid Credentials
Checks the main login flow. If this fails, users cannot access the system.
TC-002: Minimum Password Login
Validates boundary conditions for password rules.
TC-003: Forgot Password
Ensures users can recover access when they forget their password.
TC-004: Login After Password Change
Confirms the system accepts updated passwords and rejects old ones.
TC-005: Login After Clearing Session
Verifies fresh session creation works after cache/cookie reset.
TC-006: Invalid Email Format
Ensures proper form validation and reduces invalid backend requests.
TC-007: Password Too Short
Checks enforcement of password length requirements.
TC-008: Wrong Password
Ensures incorrect credentials are rejected for security.
TC-009: Locked/Disabled Account
Verifies account lockout mechanisms prevent unauthorized access.
TC-010: Empty Fields
Confirms required field validation works before submitting the form.
